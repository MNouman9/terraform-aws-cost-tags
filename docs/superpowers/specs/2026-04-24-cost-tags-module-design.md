# Design: terraform-aws-cost-tags

**Date:** 2026-04-24  
**Status:** Approved  
**Type:** Public Terraform Registry Module (FinOps / Cost Allocation)

---

## Overview

A plug-and-play Terraform module for AWS that produces a standardised cost allocation tag map with 3-level inheritance (org ‚Üí team ‚Üí resource). It provisions no AWS resources ‚Äî it only computes and outputs tag maps for consumption by `default_tags` or individual resource `tags` blocks. Ships with a Terraform Sentinel policy for local CLI enforcement in CI pipelines.

Target audience: any AWS Terraform project that needs consistent FinOps tagging without re-implementing merge logic in every codebase.

## Variables

### Required Inputs

| Variable | Type | Validation |
|---|---|---|
| `org_name` | `string` | non-empty |
| `environment` | `string` | one of: `dev`, `sit`, `staging`, `prod` |
| `project` | `string` | non-empty |
| `team` | `string` | non-empty |
| `cost_center` | `string` | non-empty |

### Optional Inputs

| Variable | Type | Default | Description |
|---|---|---|---|
| `org_tags` | `map(string)` | `{}` | Org-wide base tags, applied to everything |
| `team_tags` | `map(string)` | `{}` | Team-level tags, merged over org_tags |
| `resource_tags` | `map(string)` | `{}` | Resource-specific tags, highest priority |
| `additional_tags` | `map(string)` | `{}` | Escape hatch for one-off tags |
| `managed_by` | `string` | `"terraform"` | Value for the ManagedBy tag |

## Tag Merge Logic

Implemented entirely in `locals` ‚Äî no resources, no data sources.

```hcl
locals {
  base_tags = {
    Environment = var.environment
    Project     = var.project
    Team        = var.team
    CostCenter  = var.cost_center
    OrgName     = var.org_name
    ManagedBy   = var.managed_by
  }

  merged_tags = merge(
    local.base_tags,      # level 0: required schema (always present)
    var.org_tags,         # level 1: org-wide additions
    var.team_tags,        # level 2: team overrides
    var.resource_tags,    # level 3: resource-specific overrides
    var.additional_tags   # escape hatch (highest priority)
  )
}
```

Right side wins on key collision. A team can override `CostCenter` from org default. A resource can override `CostCenter` from team default.

## Outputs

| Output | Type | Description |
|---|---|---|
| `tags` | `map(string)` | **Primary output.** Fully merged map. Plug into `default_tags` or resource `tags`. |
| `base_tags` | `map(string)` | Required-schema tags only, no overrides. Useful for auditing. |
| `org_tags` | `map(string)` | Level 1 merged result (base + org_tags). |
| `team_tags` | `map(string)` | Level 2 merged result (base + org + team_tags). |
| `tag_keys` | `list(string)` | Sorted list of all final tag keys. Used by Sentinel mock generation. |

## Usage Patterns

### Via `default_tags` (recommended ‚Äî tags all resources automatically)

```hcl
module "cost_tags" {
  source  = "YOUR_ORG/cost-tags/aws"
  version = "1.0.0"

  org_name    = "acme"
  environment = "prod"
  project     = "payments"
  team        = "platform-team"
  cost_center = "eng-001"

  team_tags = {
    Slack    = "#platform-alerts"
    CostCode = "P-2024-Q2"
  }
}

provider "aws" {
  region = var.aws_region
  default_tags {
    tags = module.cost_tags.tags
  }
}
```

### Per-resource override

```hcl
resource "aws_lambda_function" "processor" {
  # ...
  tags = merge(module.cost_tags.tags, {
    CostCenter = "data-pipeline"
  })
}
```

## Sentinel Policy

### Goal

Fail the Terraform plan if any AWS resource being created or updated is missing one or more of the 5 required tag keys: `Environment`, `Project`, `Team`, `CostCenter`, `ManagedBy`.

### Scope

All resource types in the plan (`aws_*`). No allowlist ‚Äî any taggable resource that enters create/update state is checked.

### Enforcement Level

`hard-mandatory` ‚Äî plan fails, not just warns.

### Local CLI Workflow

```bash
terraform plan -out=tfplan.binary
terraform show -json tfplan.binary > tfplan.json
sentinel apply -config=sentinel/sentinel.hcl sentinel/policies/require-cost-tags.sentinel
```

### Unit Testing (fully offline)

```bash
sentinel test sentinel/policies/require-cost-tags.sentinel
```

Mock data covers 12 resource types (EC2, RDS, S3, Lambda, EKS node group, SQS, SNS, ElastiCache, VPC, subnet, security group, IAM role) with passing and failing variants for each.

## Viewing Costs in AWS Console (README section)

The README will include a dedicated section explaining how to see tagged costs after applying the module. It will cover three AWS-native tools:

### AWS Cost Explorer
1. Open **AWS Console ‚Üí Billing ‚Üí Cost Explorer**
2. Choose **Tag** as the Group By dimension
3. Select a tag key (`CostCenter`, `Team`, `Project`, or `Environment`) from the dropdown
4. Set the date range and choose a granularity (daily / monthly)
5. Cost Explorer shows a breakdown of spend per tag value ‚Äî e.g. all costs attributed to `Team = platform-team`

> **Note:** Tags must be activated as cost allocation tags before they appear in Cost Explorer. This is a one-time step per tag key.

### Activating Tags as Cost Allocation Tags (required first step)
1. Go to **Billing ‚Üí Cost Allocation Tags**
2. Find the tag keys (`Environment`, `Project`, `Team`, `CostCenter`, `ManagedBy`) under the **User-defined** tab
3. Select all five and click **Activate**
4. Allow up to **24 hours** for AWS to start attributing costs to the newly activated tags

### AWS Cost and Usage Report (CUR)
For programmatic access or Athena/QuickSight dashboards, users can enable a CUR and filter/group by the same tag keys. The README will point to the AWS documentation for CUR setup.

### AWS Budgets
Users can create tag-based budget alerts: **Billing ‚Üí Budgets ‚Üí Create Budget ‚Üí Filter by Tag**. This lets a team set a monthly spend threshold per `CostCenter` or `Environment` and receive SNS/email alerts when it is breached.

- No AWS resources provisioned by the root module
- No `aws` provider block in the root module (callers own their provider)
